⚡ Analisi Prestazioni JPA

Confronto tra strategie di caricamento JPA su N vini.

🐢 LAZY (default)

Carica le relazioni solo quando vengono accedute. Genera il problema N+1: 1 query per i vini + 1 per ogni produttore.

—

Query generate: N

⚡ JOIN FETCH

Carica vini e produttori in una sola query con JOIN SQL. Elimina completamente il problema N+1.

—

Query generate: 1

📊 Conclusioni

La strategia LAZY è comoda ma pericolosa in produzione: con molti record genera decine di query inutili.

JOIN FETCH risolve il problema N+1 caricando tutto in una query con JOIN SQL. È la soluzione raccomandata quando sai in anticipo che avrai bisogno delle relazioni.

Il trade-off: JOIN FETCH carica sempre tutto anche se non serve. Per casi complessi si usa @EntityGraph per specificare esattamente cosa caricare caso per caso.
